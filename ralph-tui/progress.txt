## Codebase Patterns
- Uses electron-vite for dev/build (not plain webpack or vite)
- Single tsconfig.json with noEmit for typecheck; electron-vite handles actual compilation
- ESLint flat config (eslint.config.js) with typescript-eslint (no react-hooks plugin)
- Package type is "module" (ESM)
- Source structure: src/main/ (Electron main process), src/preload/ (context bridge), src/renderer/ (React UI)
- Editor components live in src/renderer/editor/
- Run `npx tsc --noEmit` for typecheck, `npx eslint src/` for lint
- Build with `npx electron-vite build`
- Monaco Editor uses @monaco-editor/react with `key` prop for tab switching
- MATLAB language definition is in src/renderer/editor/matlabLanguage.ts
- IPC pattern: ipcMain.handle (main) → ipcRenderer.invoke (preload) → typed in env.d.ts — keep all three in sync
- File dialog extensions use no dot: ['m'] not ['.m']
- Cross-panel communication: lift shared state to App.tsx (e.g., pendingOpenPath pattern) rather than refs or events
- FileBrowser component is in src/renderer/panels/FileBrowser.tsx with full filesystem IPC
- OctaveProcessManager in src/main/octaveProcess.ts uses delimiter pattern (___MATSLOP_CMD_DONE___) to detect command completion
- Octave process IPC: octave:start, octave:execute, octave:interrupt, octave:restart, octave:getStatus + event listeners onOctaveStatusChanged, onOctaveCrashed
- OctaveEngineStatus type ('ready' | 'busy' | 'disconnected') is exported from App.tsx and used throughout
- StatusBar is outside Allotment; App uses flex column with .app-main (flex:1) wrapper around Allotment
- CursorPosition tracked via Monaco onDidChangeCursorPosition, threaded TabbedEditor → EditorPanel → App.tsx
- FileBrowser cwd is lifted to App.tsx via onCwdChange callback
- lucide-react installed for icons — use `import { IconName } from 'lucide-react'` with `size={16}` prop
- EditorToolbar sits between PanelHeader and editor-panel-content in EditorPanel
- PendingCommand pattern (with unique id counter) for App.tsx → CommandWindow external command execution
- Monaco editor instance accessible in EditorPanel via TabbedEditor's onEditorRef callback
- WorkspacePanel uses refreshTrigger counter from App.tsx (incremented in handleCommandExecuted) to refresh after each command
- Dialog overlay pattern: fixed z-index 2000, click-outside closes via e.target === e.currentTarget, Escape key via window keydown listener
- PanelVisibility in App.tsx has 4 keys: fileBrowser, workspace, commandWindow, commandHistory — update both App.tsx and EditorPanel.tsx when adding panels
- Command history persisted via IPC to app.getPath('userData')/command-history.json
- Figure capture: handleCommandExecuted sends combined pwd+figure-capture Octave script, parses __MATSLOP_PWD__ and __MATSLOP_FIG__ markers
- Right column uses nested vertical Allotment: WorkspacePanel top, FigurePanel bottom (auto-visible when figures exist)
- Figures IPC: figures:readImage (base64), figures:saveDialog, figures:copyFile — bridge methods prefixed with `figures`
- All UI colors use CSS variables (--bg-main, --text-primary, etc.) defined in styles.css :root — use variables, never hardcode colors
- Theme config: appConfig.ts (electron-store) for general settings; resolved theme ('light'|'dark') available in App.tsx as resolvedTheme
- Keyboard shortcuts centralized in src/renderer/shortcuts/shortcutManager.ts — add new shortcuts there, not as scattered keydown handlers
- Monaco built-in actions: actions.find, editor.action.startFindReplaceAction, editor.action.gotoLine, editor.action.commentLine
- Auto-complete: matlabCompletionProvider.ts has module-level state updated via updateWorkspaceVariables() and updateMFileNames() — call these to refresh completions
- Diagnostics: matlabDiagnostics.ts analyzeMatlabCode() returns diagnostics; use diagnosticsToMarkers() to convert to Monaco markers; set via monaco.editor.setModelMarkers(model, 'matlab-diagnostics', markers)

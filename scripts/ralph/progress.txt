## Codebase Patterns
- `window.matslop` is writable (regular window property), but its properties are frozen by contextBridge — use Proxy wrapping, not mutation
- The octaveBusyTracker is a module-level singleton in octaveBusyTracker.ts — App.tsx subscribes to it via useEffect
- Typecheck: `npx tsc --noEmit`, Tests: `npx vitest run`
- rc-dock caches loadTab results (PureComponent). Use React Context (AppContext) for dynamic panel state, NOT props passed through MatslopDockLayout slots
- Panel components accept optional props that override AppContext values — this allows testing without a provider
- Callbacks from App.tsx useCallback are stable and safe as frozen props in rc-dock's cache; only data values need context
- Use `runCaptureAndRefresh()` (not `setWorkspaceRefreshTrigger` directly) after any Octave command execution — it serializes capture + workspace refresh
- When extracting useCallback helpers, define them BEFORE any callbacks that reference them (block-scoping)

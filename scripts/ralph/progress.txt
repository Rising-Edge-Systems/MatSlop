- **Learnings:**
  - DnD drop zones must be visibly large during an active drag; a 4px line looks tidy but is unhittable
  - `e.currentTarget` inside a drag handler is the element that has the draggable attribute — setting inline style on it only affects the handle, not the whole cell. Prefer className-driven styling for the dragging state.
  - Cell output stays "attached" to its cell automatically because reorderCells moves the same object reference; as long as callers keep _figures/_statementResults on the cell object (not in a parallel map), outputs won't orphan after reorder
  - Playwright HTML5 DnD: use page.evaluate + dispatchEvent('dragstart'|'dragover'|'drop'|'dragend') with `new DataTransfer()` instead of locator.dragTo(); React's synthetic event system picks it up

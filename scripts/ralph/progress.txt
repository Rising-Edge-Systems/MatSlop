## Codebase Patterns
- `download-octave.js` is a plain JS ESM script (not TypeScript) — it uses `import` syntax and runs via `node scripts/download-octave.js`
- Linux wrapper script is at `resources/octave/bin/octave-cli`, real binary at `resources/octave/usr/bin/octave-cli` — octaveConfig.ts auto-discovers the wrapper
- The script uses a `download()` helper with redirect following for HTTP/HTTPS downloads
- Platform detection: `--platform=<name>` CLI arg or `process.platform`
- Typecheck is `tsc --noEmit`, tests are `vitest run`
- Tests read the source file as a string and match patterns — be aware template literals won't resolve in source text matching
- Renderer accesses platform via `window.matslop.platform`; main process uses `process.platform`
- Figure capture script is in App.tsx `runCaptureAndRefresh` (~line 1015) — print device and resolution are set there
- Ubuntu .deb URLs must be verified against the actual archive listing — versions change (e.g. 1build4→1build5)
- hasBundledOctaveBinary() tests execution, not just file existence — needed because .deb extraction requires system shared libs

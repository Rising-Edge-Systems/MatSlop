## Codebase Patterns
- Layout uses Allotment panes; panel sizes persisted via window.matslop.layoutSet
- Panels use data-testid attributes for Playwright selectors
- Run `npm run build` to verify Vite/electron-vite production build
- Pre-existing typecheck errors in src/main/octaveProcess.ts (env.PATH) are baseline noise
- Integration/unit tests that need bundled Octave fail in dev envs without resources/octave
- Prefer extracting pure helpers into src/renderer/editor/editorTypes.ts and adding vitest unit tests in tests/unit/
- For HTML5 drag/drop E2E: dispatch DragEvents manually via page.evaluate with a shared DataTransfer — Playwright's native drag is flaky
- LiveScript cell data lives in LiveScriptEditor.tsx; serialized via parseLiveScript/serializeLiveScript in editorTypes.ts
- `.ls-cells-grid` is a 2-column CSS grid; each code cell emits paired sibling grid items (code-side at col 1, output-side at col 2). Full-width rows (drop zones, add-cell rows, markdown cells) use `grid-column: 1 / -1`. Hover highlight is driven by a shared `hoveredCellId` React state that applies `.ls-cell-hover` to both halves.
- Per-cell controls (gutter = drag handle + run button/spinner; actions = delete button) live inside `.ls-cell-code-side` (code column), not on the output side. `.ls-cell-actions` uses `opacity: 0` + hover to fade in, so Playwright clicks need `{ force: true }`.
- Live-script vertical scrolling is container-level on `.ls-editor` (overflow-y: auto). Per-cell `.ls-cell-output` uses `max-height: none; overflow-y: visible; overflow-x: auto` so tall outputs expand their grid row (the container scrolls) and wide text scrolls inside the cell. `.ls-cell-output-side` uses `overflow-x: hidden` to prevent wide content from widening its `minmax(0,1fr)` grid track and breaking row alignment.
- Bundled Octave helper scripts live in `resources/octave-scripts/*.m`. When wiring them to the running Octave process, add the directory to `extraResources` in package.json AND addpath it at Octave startup (or copy into the bundled site m/ path).
- Plot JSON schema lives in `src/main/plotSchema.ts`. Use `parsePlotFigure(json)` to validate; unknown series types degrade to `{type:'unknown', octaveType}` so UI can show fallbacks. Emit JSON in Octave with `matslop_export_fig(h)` — it ships its own JSON encoder (don't assume `jsonencode` exists).

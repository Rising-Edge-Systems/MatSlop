## Codebase Patterns
- rc-dock caches React elements in loadTab via PureComponent. ALL props passed through dock slots are frozen at mount time.
- The ONLY way to pass dynamic data to dock panels is React Context (useAppContext / useOctaveStatus). Props are stale.
- Stable callbacks: useCallback with [] deps, or useCallback that reads from refs internally. These are safe as dock props.
- Unstable callbacks: useCallback with state in deps like [tabs, octaveStatus, callStack]. These MUST be converted to ref-based.
- The ref-stabilization pattern: `const fooRef = useRef(foo); fooRef.current = foo;` then read fooRef.current inside useCallback with [] deps.
- AppContext.tsx holds dynamic state. OctaveContext.tsx holds engineStatus. Both bypass rc-dock caching.
- Panel components use `const ctx = useAppContext(); const val = ctx.field ?? propField;` pattern for context+prop fallback.
- octaveEngineStatusRef already exists in App.tsx for stable access to engine status.
- tabsRef, activeTabIdRef already exist in EditorPanel.tsx.
- Tests render panels without providers, so props must remain as optional fallbacks.

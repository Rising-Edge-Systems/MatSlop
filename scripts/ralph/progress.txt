## Codebase Patterns
- download-octave.js: per-platform pinned versions (WIN_OCTAVE_VERSION, MAC_OCTAVE_VERSION). Windows ships from ftp.gnu.org/gnu/octave/windows (NOT ftpmirror.gnu.org — that redirector 502s). macOS is pinned to octave-app v9.2 (release ceiling). Linux is intentionally NOT bundled — no maintained AppImage source; the script exits cleanly with package-manager guidance and src/main/octaveConfig.ts handles PATH lookup at runtime.
- Tests that assert download-octave.js content live at tests/unit/download-octave-{linux,mac}.test.ts and read the script as a string — any URL or constant change needs matching regex updates there.
- octaveConfig.ts retains its legacy squashfs-root branch as a fallback for users who manually drop an AppImage into resources/octave/, even though the download script no longer populates it.
- Renderer-side octaveExecute interception: wrap `window.matslop.octaveExecute` once from src/renderer/main.tsx (before any component mounts) via an idempotent shim. Avoids touching ~30 call sites across panels/dialogs. `src/renderer/octaveBusyTracker.ts` is a working example — it ref-counts in-flight executes for the "Running…" status-bar indicator.
- Use `vi.advanceTimersByTimeAsync` (not the sync variant) when code under test awaits promises between timer fires — the async version drains microtasks between ticks.
- To unit-test OctaveProcessManager without a real child, mock `child_process` at module level (`vi.mock('child_process', () => ({ spawn: vi.fn(() => new FakeChildProcess()) }))`) then `await import('.../octaveProcess')` AFTER the mock. FakeChildProcess extends EventEmitter with `stdin.write`, `stdout`, `stderr` emitters; emit `___MATSLOP_CMD_DONE___\n` on stdout to complete the init handshake.
- OctaveProcessManager.interrupt() (SIGINT, keeps process) and .stop() (writes 'exit' then SIGKILLs after 2s) are NOT interchangeable. The editor Stop button calls interrupt(); only full shutdown/restart uses stop().

- **Learnings for future iterations:**
  - To unit-test OctaveProcessManager without spawning real Octave: `vi.mock('child_process', () => ({ spawn: vi.fn(() => new FakeChildProcess()) }))` where FakeChildProcess extends EventEmitter and exposes stdin.write / stdout / stderr event emitters. Then import OctaveProcessManager AFTER the mock with `await import(...)` so the manager picks up the stubbed spawn.

Top-level prompt commands still cancel normally. If future polish needs a stricter "cancel no matter what" Stop, consider queuing a `dbquit` after SIGINT.

- **Learnings for future iterations:**
  - JS `Number.prototype.toPrecision(6)` only flips to exponential when |n|<1e-6 by default — if you want "1e-5" instead of "0.00001" you must branch on magnitude and call `toExponential(5)` manually.

Getting this order wrong makes a 1x1 struct render as a bogus number.
